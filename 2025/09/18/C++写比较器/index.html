优先队列的模板定义

在C++的std::priority_queue（优先队列）中，第三个模板参数是比较器（Comparator），它用于定义元素的优先级规则。

std::priority_queue的完整模板声明如下：

  • 第一个参数T：队列中元素的类型。
  • 第二个参数Container：底层容器（默认是std::vector），需支持push_back、pop_back、front等操作。
  • 第三个参数Compare：比较器，用于确定元素的优先级。默认是std::less<T>，此时优先队列默认是最大堆（优先级最高的元素是最大的）。

比较器的作用

比较器的核心作用是定义“如何判断两个元素的优先级高低”。
对于比较器Compare，当Compare(a, b)返回true时，表示a的优先级低于b，因此b会被优先弹出（即b排在更前面）。

如何实现比较器？

比较器可以通过以下几种方式实现：

1. 使用标准库自带的比较器

C++标准库提供了两个常用的比较器：

  • std::less<T>：默认比较器，实现最大堆（大元素优先）。
  • std::greater<T>：实现最小堆（小元素优先）。

lambda表达式可以简化比较器的定义，但需要配合decltype推导类型。

关键注意点

  • 比较器的返回值逻辑：compare(a, b) == true 表示a优先级低于b（b先出队）。
  • 自定义比较器需满足“严格弱序”（strict weak ordering），否则可能导致未定义行为。
  • 底层容器必须支持随机访问迭代器（如vector、deque），不能用list（不支持随机访问）。

通过自定义比较器，你可以灵活控制优先队列中元素的优先级规则，适应不同的业务场景。

通过定义结构体（或类）并重载operator()来实现自定义比较器

通过定义结构体（或类）并重载operator()来实现自定义比较器，是C++中最常用也最灵活的方式之一。这种方式本质是创建一个函数对象（Functor）——即可以像函数一样被调用的对象，通过它来定义优先队列中元素的优先级规则。

优先队列在判断元素优先级时，会调用比较器的operator()方法。对于两个元素a和b：

  • 若comparator(a, b)返回true，表示a的优先级低于b，因此b会被优先弹出（排在队列更前面）；
  • 若返回false，表示a的优先级不低于b，因此a可能排在更前面。

实现步骤

1. 定义结构体（或类）

通常使用结构体（struct）更简洁，因为结构体默认成员是公有的，无需额外声明访问权限。

2. 重载operator()

在结构体中重载operator()，函数签名通常为：
bool operator()(const T& a, const T& b) const;

  • 参数：两个待比较的元素a和b（建议用const引用，避免拷贝开销）；
  • 返回值：bool类型，表示a是否优先级低于b；
  • const修饰：确保比较器对象在常量上下文中也能使用。

在 bool operator()(const T& a, const T& b) const; 中，最后的 const 是成员函数的常量性修饰符，用于限定这个 operator() 成员函数的行为。它的核心作用是：

  1. 表明函数不会修改类的非静态成员变量
    当一个成员函数被 const 修饰时，编译器会强制检查函数内部是否有修改类的非静态成员变量的操作。如果有，则会编译报错。
    这是一种编译期的约束，确保该比较器在执行比较逻辑时，不会意外修改自身的状态（比如比较器中可能存在的配置参数）。

  2. 允许在 const 对象 /const 引用上调用该函数
    C++ 中，const 对象或 const 引用只能调用其 const 成员函数。如果 operator() 没有最后的 const，则无法在 const 上下文中使用这个比较器。
    而标准库的 priority_queue 在内部可能会通过 const 引用使用比较器，因此要求比较器的 operator() 必须是 const 成员函数，否则会导致编译错误。

3. 在优先队列中指定比较器

将自定义结构体作为std::priority_queue的第三个模板参数。

关键注意事项

  1. 严格弱序（Strict Weak Ordering）
    比较器必须满足“严格弱序”，否则优先队列的行为会是未定义的（可能崩溃、排序错误等）。
    严格弱序的核心要求：

    • 非自反：comparator(a, a)必须返回false（一个元素不能比自己优先级低）；
    • 反对称：若comparator(a, b)为true，则comparator(b, a)必须为false；
    • 传递性：若comparator(a, b)和comparator(b, c)为true，则comparator(a, c)必须为true。

    例如，若比较器写为return a >= b，则违反非自反性（a >= a为true），会导致错误。

  2. 参数的const与引用
    比较器的operator()参数建议用const T&（常量引用），原因：

    • 避免不必要的对象拷贝，提高效率；
    • 确保比较过程中不会修改原对象（逻辑更安全）。

  3. 复用性
    结构体比较器可以在多个优先队列中复用，也可以作为其他需要比较逻辑的场景（如std::sort）的参数，比lambda表达式的复用性更强。

通过这种方式，你可以实现任意复杂的优先级规则，满足各种业务场景（如任务调度、事件处理、路径规划等）的需求。
